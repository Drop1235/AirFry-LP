# AirFry シークレットセール用ゲート機能 設計ドキュメント（本番LINE連携版）

## 1. 目的と概要
本ドキュメントは、「AirFry販売サイト」へアクセスしたユーザーに対し、「LINE友だち」および「メールアドレス登録」を促すためのゲート（アクセス制限）機能を、`Junior-Portal-Web`の実装コードを流用した本物のLINEログイン機能を用いて実装するための設計仕様を定義するものです。
この設計に基づき、1つのファイル（`index.html`）内でLIFF APIを利用した状態遷移を実現します。

## 2. デザイン・配色の要件
*   **カラースキーム**: 「白・ブルー基調」（健康的・信頼性・アクティブ）。ブラック系は避ける。
*   **UIスタイル（上段/訴求）**: 信頼感を与えるため、薄いブルーグレーの背景にドクターズパネル風のクリーンなリストレイアウトを採用。
*   **UIスタイル（下段/魅力）**: 先進的でアクティブな印象を与えるため、白背景のカード型を採用し、美しいシャドウとホバー時の浮き上がりなどインタラクティブな要素で強調する。
*   **アクセントカラー**: LINEログインボタンは公式カラー `#06C755`。メインのアクセントはブルー。

## 3. UI/UXフロー設計（3ステップの状態遷移）と LINEログイン本番連携設計

`Junior-Portal-Web`のLIFF実装（LIFF ID: `2009206725-w4u2ipHo`）を流用し、実際のLINE認証を利用した表示切り替えを行います。

### 導入: LIFF初期化処理
*   ページロード時にLINE公式のSDK (`https://static.line-scdn.net/liff/edge/2/sdk.js`) を読み込み、`liff.init()` を実行。
*   `liff.isLoggedIn()` を判定し、未ログインの場合はステップ1を表示。ログイン済みの場合はステップ2を表示。

### ステップ1：LINEログイン画面（本番連携）
*   **状態**: ユーザーがLINEで未ログインの場合に表示（メインコンテンツ・ステップ2は非表示）。
*   **文言**: 
    「テンプマチへようこそ
    本サイトは「ゼントリ公式アカウント」
    友だち限定のサービスです。
    LINEでログインしてご利用ください。」
*   **アクション**: 「LINEでログイン」ボタンを押下すると、`liff.login({ redirectUri: window.location.href })` が実行され、実際のLINE認証画面へリダイレクトされる。
*   **友だちチェック連携**: `Junior-Portal-Web` に準拠し、未友だち時の処理を含める場合は `liff.getFriendship()` を利用する（本実装は認証成功状態を担保するため `isLoggedIn` 判定を必須とする）。

### ステップ2：メールアドレス登録画面（LINE認証成功後に表示）
*   **状態**: LIFF認証のリダイレクトから戻り、`liff.isLoggedIn() === true` となった場合に初めて表示される。
*   **マーケティング訴求**: 「メールアドレスをご登録いただくと、AirFryを【限定特別価格】でご購入いただけます！」
*   **入力項目**: メールアドレス入力欄。
*   **同意項目**: 指定同意文言およびチェックボックス（必須）。
*   **アクション**: 「同意して特別価格で購入する」ボタン押下により、入力値を取得し、ステップ3へ状態遷移する。

### ステップ3：メインコンテンツ解放とプレフィル連携
*   **状態**: ステップ2のフォーム送信後、ゲート表示全体を非表示にしメインコンテンツを表示。
*   **プレフィル連携**: 入力されたEメールをパラメータとしてGoogleフォームのURLに結合し、別タブで開く注文ボタンの href 属性に自動反映させる。

## 4. データフロー設計（プレフィル連携）
1.  ステップ2で「同意して特別価格で購入する」押下時に入力メアド（`example@example.com`）を取得。
2.  GoogleフォームURLに以下のパラメータを動的付与：
    *   `?embedded=true&usp=pp_url&entry.1788641189={入力メアド}`
3.  生成したURLを、メインコンテンツ下部の `<a id="order-button">` の `href` 属性にセットする。
4.  （オプション）画面リロード時に毎回入力を求められないよう、`localStorage` に `emailRegistered` フラグとメールアドレスを保持し、次回以降は自動でステップ3へ進めるフォールバックも組み込む。

## 5. コンポーネント構造・DOM定義
```html
<body>
  <div id="gate-container">
    <!-- ローディング表示（LIFF初期化中） -->
    <div id="step-loading">読み込み中...</div>

    <!-- ステップ1: LINEログイン (未ログイン時) -->
    <div id="step1-login" style="display: none;">
      <h2>AirFry公式アカウント 友だち限定サービス</h2>
      <button onclick="loginWithLine()">LINEでログイン</button>
    </div>

    <!-- ステップ2: メール登録 (ログイン済・メアド未登録時) -->
    <div id="step2-email" style="display: none;">
      <p>メールアドレスをご登録いただくと、AirFryを【限定特別価格】でご購入いただけます！</p>
      ...
    </div>
  </div>

  <!-- ステップ3: メインコンテンツ (ログイン済・メアド登録完了後) -->
  <div id="main-content" style="display: none;">
    ...
    <a id="order-button" href="#" target="_blank" rel="noopener noreferrer">特別価格で注文する（別ウィンドウで開きます）</a>
  </div>
</body>
```
上記設計に基づいて作成することで、本物のLINEログインフローと完全連携したゲートシステムとなります。
