# AirFry シークレットセール用ゲート機能 設計ドキュメント

## 1. 目的と概要
本ドキュメントは、「AirFry販売サイト」へアクセスしたユーザーに対し、「LINE友だち」および「メールアドレス登録」を促すためのゲート（アクセス制限）機能のフロントエンド設計仕様を定義するものです。
この設計に基づき、1つのファイル（`index.html`）内でCSS/JavaScriptを完結させ、3ステップの状態遷移を実現します。

## 2. デザイン・配色の要件
*   **カラースキーム**: 「白・ブルー基調」とし、健康的・信頼性・アクティブを感じさせるデザインとします。ブラック系は避けます。
*   **UIスタイル**: カード型デザイン（白背景に薄い影）を採用し、シンプルで直感的な操作が可能なUIとします。
*   **アクセントカラー**: LINEログインボタンは公式カラーである `#06C755` を使用し、購入意欲を高めるボタンにはメインのアクセントブルー（ `#007aff` ）などを活用します。

## 3. UI/UXフロー設計（3ステップの状態遷移）
初期状態ではメインコンテンツ（商品紹介・注文シミュレーターなど）を非表示とし、全画面のゲートコンテナを表示します。

### ステップ1：LINEログイン画面
*   **状態**: メイン・コンテンツ非表示（`display: none;`）。ゲート用コンテナ内で本画面を表示。
*   **文言**: 「AirFry公式アカウント 友だち限定サービス」
*   **アクション**: 「LINEでログイン」ボタン（緑色: `#06C755`）を押下することで、ステップ2へ遷移する（JavaScriptによる要素の表示切替）。

### ステップ2：メールアドレス登録画面
*   **状態**: ステップ1の画面を非表示（`display: none;`）にし、本画面を表示（`display: block;`）。
*   **マーケティング訴求テキスト**: 「メールアドレスをご登録いただくと、AirFryを【限定特別価格】でご購入いただけます！」
*   **入力項目**: メールアドレス入力欄（`type="email"`、必須）。
*   **同意項目**:
    *   【指定同意文言】: 「ご登録いただいたメールアドレス宛に、AirFryの限定価格でのご案内やシークレットセールの情報をお送りします。また、弊社が提供・取り扱う各種サービスや商品に関する特別なお知らせも合わせてお届けいたします。」
    *   チェックボックス（必須）。ONになった場合のみ次へ進める。
*   **アクション**: 「同意して特別価格で購入する」ボタンを押下することで、入力されたメールアドレスを取得し、ステップ3へ遷移する。

### ステップ3：メインコンテンツ解放とプレフィル連携
*   **状態**: ゲート用コンテナ全体を非表示（`display: none;`）にし、メインコンテンツ（`#main-content` など）を表示（`display: block;` あるいはフェードイン）。
*   **アクション**: 表示切替と同時に、下部のGoogleフォーム（iframe）の `src` 属性を動的に書き換え、メールアドレスをプレフィル（自動入力）反映する。

## 4. データフロー設計（プレフィル連携）
Junior-Portal-Webの実装を参考に、取得した入力値をフォームURLへ結合し反映します。

1.  ステップ2のフォーム送信時（またはボタン押下時）、JavaScriptで入力されたメールアドレスの文字列（例：`test@example.com`）を取得。
2.  GoogleフォームのベースURLに対して、URLパラメータを追加して結合する。
    *   **ベースURL**: `https://docs.google.com/forms/d/e/1FAIpQLSeeDvsQ9k-AqJ6x_UDqSOxVRlSTxLSskrqUc2DHyXv-R-6iqQ/viewform`
    *   **付与パラメータ**: `?embedded=true&usp=pp_url&entry.1788641189=`
3.  結合したURL（`.../viewform?embedded=true&usp=pp_url&entry.1788641189=test@example.com`）を、対象の iframe の `src` 属性にセットする。
4.  これにより、ユーザーがメイン画面下部の注文フォームへ到達した際、既にメールアドレスが入力された状態となる。

## 5. コンポーネント構造・DOM定義

以下は `index.html` 内の主要なDOM構造の概要です。

```html
<body>
  <!-- ゲートコンテナ (初期表示) -->
  <div id="gate-container">
    
    <!-- ステップ1: LINEログイン -->
    <div id="step1-login" class="gate-card">
      <h2>AirFry公式アカウント 友だち限定サービス</h2>
      <button onclick="goToStep2()">LINEでログイン</button>
    </div>

    <!-- ステップ2: メール登録 -->
    <div id="step2-email" class="gate-card" style="display: none;">
      <p>メールアドレスをご登録いただくと、AirFryを【限定特別価格】でご購入いただけます！</p>
      <input type="email" id="user-email" required>
      <p class="terms">ご登録いただいたメールアドレス宛に、...</p>
      <input type="checkbox" id="agree-check">
      <button onclick="submitEmail()" id="btn-submit" disabled>同意して特別価格で購入する</button>
    </div>

  </div>

  <!-- メインコンテンツ (初期非表示) -->
  <div id="main-content" style="display: none;">
    <!-- ヘッダー、ヒーロー、説明セクション等 (既存コンテンツ) -->
    
    <!-- 注文フォームセクション -->
    <iframe id="order-form-iframe" src="..."></iframe>
  </div>

  <!-- JavaScriptロジック -->
  <script>
    // 状態遷移管理、チェックボックスの監視、iframe src の動的書き換えロジックを含める
  </script>
</body>
```
以上の設計定義に基づき、ステップ1からステップ3までの一連のフローとデータ連携を実装します。
